# Pacheco AI ( In development!)
Pacheco AI is a self-healing, LLM-driven development agent designed to bridge the gap between AI-generated logic and local execution. Operating under the "LIMA CORP" methodology, it utilizes a dual-mode logic gate (Developer/Architect) to write, test, and repair Python scripts in real-time within a secure sandbox environment.

🛠 Prerequisites & Installation
To deploy the Pacheco Ω system and achieve peak performance, follow these technical requirements:

1. Python Environment
Version: Python 3.10+ (Recommended)

Why? We leverage advanced type hinting, modern subprocess handling, and the latest requests session management for low-latency communication with the LLM.

2. Dependencies
Install the required libraries using the provided requirements.txt:

Bash
pip install -r requirements.txt
requirements.txt content:

requests: For robust HTTP communication with the Ollama API.

datetime: (Standard Library) For timestamped execution logs.

re: (Standard Library) For advanced regex-based code block extraction.

json: (Standard Library) For state management and data persistence.

3. Local LLM Setup (The "Brain")
Pacheco Ω currently runs on Phi-3 (3.8b), optimized for high-speed reasoning and minimal hallucination in Python syntax.

Download Ollama: ollama.com

Pull the Model:

Bash
ollama pull phi3:3.8b
Run the Server: Ensure the Ollama service is running locally on http://localhost:11434.

📂 System Architecture & Workspace
The system is designed with a Zero-Leakage Security Protocol. It automatically initializes and manages two primary directories:

1. /Sandbox
Purpose: The testing ground. Every script generated by Pacheco is first saved here with a unique timestamp (exec_1714495200.py).

Validation: The system executes the code here and captures stdout or stderr. If a NameError or SyntaxError occurs, the Self-Healing Loop triggers immediately.

2. /Projetos_Ativos
Purpose: The production-ready vault.

Deployment: Only scripts that pass the Sandbox execution without errors are promoted and saved here. These are cleaned, validated, and ready for integration.
