Pacheco AI v31.4 | LIMA CORP — Evolutionary AI System
by Arthur Lima
AI Engine Upgrade
Replaced phi3:3.8b with qwen2.5-coder:7b — a model purpose-trained for code generation. This eliminated recurring issues from the previous engine: Spanish variable names (productos, costo), functions defined but never called, and variable name typos. Code quality improved significantly, with most tasks now succeeding on the first attempt.
Advanced Security Analysis
Integrated an AST-based (Abstract Syntax Tree) pre-execution analyzer that inspects generated code structurally — without running it — blocking dangerous calls such as os.system, subprocess, eval, and unauthorized imports before they can execute.
Dual-Tier Memory System

Short-Term: Maintains the last 5 interactions in RAM for current session context.
Long-Term (JSON): Persists successful solutions to a local JSON file across sessions, with Jaccard similarity matching to reference relevant past solutions.
Memory retrieval is now selective — only activates for file and data-related tasks, preventing irrelevant solutions from contaminating unrelated prompts.

Self-Healing Retry Loop
Autonomous retry logic captures execution errors and re-prompts the model with the full error context and the failing code for automatic correction, up to 3 attempts per task.
Execution Sandbox
Dedicated sandbox environment with strict 15s timeout, automatic cleanup of temporary execution files, guaranteed UTF-8 encoding via sys.executable and environment flags, and debugger noise filtering for clean output.
Integrated Logger
Sessions are automatically logged with timestamps and status levels (SUCESSO, ERRO, FATAL) for easier debugging and progress tracking.
Strict Output Validation
The system now rejects code that executes without producing output — mandatory print() calls are enforced before confirming success, eliminating false positives.
Prompt Engineering
System prompt rebuilt with explicit rules, visual right/wrong examples, blocklist of forbidden Spanish words with correct Portuguese equivalents, and mandatory function-call enforcement. Project root directory is injected automatically at runtime.
Architecture
Fully refactored into single-responsibility classes: AnalisadorSeguranca, Executor, Memoria, PromptBuilder, ClienteOllama, Logger, PachecoAI.
